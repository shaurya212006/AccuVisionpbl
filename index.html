AccuVision

AI-based Smartphone Eye Power Screening System

"We hypothesize that combining smartphone camera imaging with deep learning-based pupil segmentation and optical modeling can create a reliable screening tool that estimates refractive errors with an accuracy of ±0.25 diopters under ideal conditions, making preliminary eye screening accessible to billions."

About This Project

  • Smartphone app that estimates eye power (spherical equivalent) using camera + AI + optics.
  • Designed for adults (18–65 years) as a quick pre‑check before visiting an eye care professional.
  • Internal simulations target ±0.25D agreement with clinical autorefractors under ideal conditions.
  • Academic prototype (v0.1.0-pre-dev); clinical validation and regulatory approvals are pending.
Screening Tool Only – Not a Diagnostic Device

Problem Statement

Uncorrected refractive errors affect billions of people worldwide, reducing productivity, education outcomes, and quality of life. Regular eye examinations require clinical equipment, trained staff, and physical visits, which are often inaccessible or expensive in rural and low‑resource settings.

  • Limited Access: Few trained eye‑care professionals in remote areas.
  • High Cost: Autorefractors and diagnostic devices cost thousands of dollars.
  • Low Awareness: Many people live with blur without realizing they need correction.
  • Time Constraints: Busy schedules and travel make routine checkups rare.

→ There is a need for a low‑cost, smartphone‑based, preliminary screening solution.

Literature Review / Market Research

Existing Solutions:

  • Traditional autorefractors (clinic‑grade, expensive, non‑portable).
  • Hand‑held screening devices (Plusoptix, 2WIN) that still require specialised hardware.
  • Consumer apps that often depend on external attachments or lack transparent accuracy data.
  • Research prototypes using CNNs / U‑Net for iris and pupil segmentation with good performance.

Vision care is a multi‑billion‑dollar global market, while smartphone penetration continues to grow, especially in emerging economies.

Research Gap / Innovation

Our Unique Approach:

  • ✓ Zero Extra Hardware: Works with a standard smartphone camera.
  • ✓ Hybrid AI + Optics: U‑Net segmentation combined with an optical model.
  • ✓ Multi‑Meridian Strategy: Simulated ~50% reduction in noise through averaging.
  • ✓ Temporal 1D CNN: Uses 30‑frame sequences to refine the optics‑only estimate.
  • ✓ Device Calibration: Per‑device constants to adapt the same model to different phones.

Dataset / Input

  • Training: Synthetic eye images for segmentation and refraction; MNIST for ML pipeline sanity checks.
  • Augmentation: Rotation, scaling, brightness, and noise to simulate real capture conditions.
  • Runtime Input: 256×256 eye crops captured from the smartphone camera.
  • Capture Setup: 3–5 images per eye at ~20 cm distance with basic alignment guidance.
  • Data Handling: Prototype assumes local processing and secure storage of summaries in Firestore.

Model / Architecture

  • U‑Net: Pupil segmentation network (Dice ≈ 0.87 on synthetic dataset).
  • MNIST CNN: Classification model used to validate the ML training/inference pipeline (97.2% accuracy).
  • Regression: Baseline refraction regressor (MAE ≈ 0.18D) for comparison.
  • 1D CNN: Temporal model over 30‑frame sequences (MAE ≈ 0.14D).
  • Optics: 0.37 constant for pixel–diopter mapping and simulated multi‑meridian averaging.

Tech Stack

  • Flutter (cross‑platform mobile app).
  • TensorFlow / Keras + TFLite for on‑device inference.
  • Flask REST API backend (Python).
  • Firebase Auth + Firestore for auth and history.
  • OpenCV, NumPy, Pandas for image and data processing.
VIEW CODE / DEMO

Live Execution Flow

  1. 1. User Onboarding: User logs in via Firebase Auth and receives brief positioning instructions and device calibration info.
  2. 2. Image Capture: Camera captures 3–5 eye frames with real‑time feedback on lighting and alignment.
  3. 3. On‑Device Processing: TFLite U‑Net segments the pupil, features are extracted, then passed through the optics model and 1D‑CNN.
  4. 4. Result Display: App shows estimated spherical equivalent (e.g., −2.25D) with a confidence score and interpretation text.
  5. 5. History Tracking: Summarised results and metadata are stored in Firestore; trends can be viewed in the app.
  6. 6. Professional Referral: Based on screening results, the app suggests booking a comprehensive eye examination.

Key Achievements

  • ✓ Target ±0.25D accuracy achieved in internal simulations (not yet clinically validated).
  • ✓ 1D‑CNN reduces MAE compared to the regression baseline (0.14D vs 0.18D).
  • ✓ Multi‑meridian fusion cuts simulated noise variance by about half.
  • ✓ All core models converted to TFLite with significantly reduced size.
  • ✓ End‑to‑end pipeline from camera capture to cloud history implemented.

Next Steps: Phase 0

  • → Conduct clinical pilot study with 50–100 volunteers comparing against an autorefractor.
  • → Collect real‑world datasets and retrain/validate all models.
  • → Finalise regulatory documentation and prepare for DCGI submission.
  • → Maintain Design History File and risk analysis (20+ identified hazards).
  • → Implement ethics board approval and informed consent workflows.

Limitations: Current accuracy estimates are based on synthetic and engineering data and assume ideal conditions (good lighting, stable positioning, fixed distance). Astigmatism and higher‑order aberrations are not modelled (spherical equivalent only). Clinical validation is pending, and the prototype is not suitable for paediatric or complex ocular pathology use cases.
